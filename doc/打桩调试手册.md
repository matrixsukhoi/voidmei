# VoidMei 打桩（Mock）调试手册

> **作者**: VoidMei 核心贡献者
> **版本**: 2.0 (2025年更新)
> **目标读者**: 对 Java 熟悉但可能对 Python 不太了解的开发者

---

## 1. 为什么要"打桩"？

在开发 VoidMei 的 HUD 布局或逻辑时，如果每次都需要启动《战争雷霆》（War Thunder）并进入试飞，效率极低且极度消耗资源。"打桩"技术通过一个轻量级的 Python 脚本，模拟游戏 8111 端口的行为，让 VoidMei 认为你正在飞行，从而实现**脱离游戏开发**。

### 工作原理

```
┌─────────────────────────────────────────────────────────────────────┐
│                         数据捕获阶段                                  │
│  War Thunder (8111) ───HTTP GET───> mock_8111.py capture            │
│                                           │                          │
│                                           ↓                          │
│                                    mock_data.json                    │
└─────────────────────────────────────────────────────────────────────┘
                                           │
                                           ↓
┌─────────────────────────────────────────────────────────────────────┐
│                         模拟服务阶段                                  │
│  mock_data.json ───读取───> mock_8111.py serve (8112)               │
│                                           │                          │
│                                           ↓ HTTP                     │
│                                    VoidMei Java App                  │
│                                    (连接 8112 端口)                   │
└─────────────────────────────────────────────────────────────────────┘
```

### 核心优势

| 特性 | 说明 |
|------|------|
| **零游戏依赖** | 不需要启动 War Thunder |
| **快速迭代** | 修改代码后立即看到效果 |
| **可复现测试** | 相同的 mock_data.json 产生相同的结果 |
| **自定义数据** | 可手动修改 JSON 测试极端情况 |

## 2. Python 环境准备

脚本位于 `script/mock_8111.py`，支持两种运行方式：

### 方案 A：使用 uv（推荐，最快）

如果你安装了 `uv`，无需关心依赖安装，直接运行即可：

```bash
# 启动 mock 服务（监听 8112 端口）
uv run script/mock_8111.py serve --port 8112
```

### 方案 B：使用原生 pip

```bash
# 1. 安装依赖
pip install requests psutil

# 2. 运行脚本
python script/mock_8111.py serve --port 8112
```

> **端口统一约定**：本文档始终建议使用 **8112** 端口，避免与真实游戏进程（8111）冲突。

## 3. 打桩脚本使用说明

### 3.1 捕获数据 (`capture`)

在**游戏运行且正在飞行**时运行，将当前飞行状态保存到 `script/mock_data.json`：

```bash
uv run script/mock_8111.py capture
# 或
python script/mock_8111.py capture
```

捕获的数据包括：
- `/state` - 飞行状态数据（高度、速度、发动机参数等）
- `/indicators` - 仪表数据（姿态、罗盘、燃油等）
- `/map_obj.json` - 地图对象（飞机、坦克、轰炸点等）
- `/map_info.json` - 地图元信息（网格大小、边界等）

### 3.2 启动模拟 (`serve`)

启动 HTTP 服务器，模拟游戏的 8111 端口行为：

```bash
uv run script/mock_8111.py serve --port 8112
# 或
python script/mock_8111.py serve --port 8112
```

启动后，VoidMei 连接到 `127.0.0.1:8112` 即可获取模拟数据。

> **提示**：mock 服务会实时读取 `mock_data.json`，修改文件内容后无需重启服务。

## 4. VoidMei 配置调整

为了让 VoidMei 连接到 Mock 服务，需要修改端口配置。

### 4.1 UI 界面配置（推荐）

1. 启动 VoidMei
2. 进入 **核心设置** → **调试** 分组
3. 找到 **8111端口** 输入框
4. 将值从 `8111` 改为 `8112`
5. 点击 **保存位置/配置**，程序会自动重新连接

### 4.2 配置文件手动修改

直接编辑 `ui_layout.cfg`：

```clojure
(group "调试"
  ;; 将 value 改为 8112
  (item "8111端口" :type input :target "httpPort" :value "8112" ...)
)
```

## 5. mock_data.json 结构参考

Mock 数据文件包含四个端点的完整响应数据：

### 5.1 `/state` - 飞行状态

| 字段 | 类型 | 说明 | 示例值 |
|------|------|------|--------|
| `valid` | boolean | 数据有效性 | `true` |
| `IAS, km/h` | number | 指示空速 | `474` |
| `TAS, km/h` | number | 真空速 | `454` |
| `H, m` | number | 高度（米） | `46` |
| `M` | number | 马赫数 | `0.39` |
| `Ny` | number | 过载（G值） | `0.35` |
| `Vy, m/s` | number | 垂直速度 | `-7.3` |
| `AoA, deg` | number | 攻角 | `-1.6` |
| `AoS, deg` | number | 侧滑角 | `-5.9` |
| `Mfuel, kg` | number | 当前燃油 | `197` |
| `Mfuel0, kg` | number | 初始燃油 | `734` |
| `throttle 1, %` | number | 油门位置 | `110` |
| `RPM 1` | number | 发动机转速 | `3001` |
| `power 1, hp` | number | 功率（马力） | `1597.8` |
| `manifold pressure 1, atm` | number | 进气压 | `2.24` |
| `water temp 1, C` | number | 水温 | `121` |
| `oil temp 1, C` | number | 油温 | `90` |
| `thrust 1, kgs` | number | 推力（喷气机） | `840` |
| `aileron, %` | number | 副翼位置 | `-48` |
| `elevator, %` | number | 升降舵位置 | `20` |
| `rudder, %` | number | 方向舵位置 | `-47` |
| `flaps, %` | number | 襟翼位置 | `0` |
| `gear, %` | number | 起落架位置 | `0` |

> **多引擎飞机**：字段后缀 `1`, `2`, `3`, `4` 表示不同引擎，如 `power 2, hp`。

### 5.2 `/indicators` - 仪表数据

| 字段 | 类型 | 说明 | 示例值 |
|------|------|------|--------|
| `valid` | boolean | 数据有效性 | `true` |
| `type` | string | 飞机型号标识符 | `"p-51d-20_china"` |
| `army` | string | 军种 | `"air"` |
| `speed` | number | 速度表读数 | `131.0` |
| `aviahorizon_roll` | number | 地平仪横滚角 | `-40.5` |
| `aviahorizon_pitch` | number | 地平仪俯仰角 | `0.63` |
| `compass` | number | 罗盘航向 | `164.1` |
| `altitude_hour` | number | 高度表（小时针） | `151.0` |
| `vario` | number | 升降速度表 | `-7.3` |
| `rpm` | number | 转速表 | `3001.5` |
| `manifold_pressure` | number | 进气压表 | `2.24` |
| `fuel1` | number | 燃油表 | `196.8` |
| `throttle` | number | 油门杆位置 (0-1.1) | `1.1` |
| `gears` | number | 起落架状态 | `0.0` |
| `flaps` | number | 襟翼状态 | `0.0` |

### 5.3 `/map_obj.json` - 地图对象

数组格式，每个元素表示一个地图对象：

```json
{
    "type": "aircraft",       // 对象类型
    "color": "#faC81E",       // 颜色（十六进制）
    "color[]": [250, 200, 30],// 颜色（RGB数组）
    "blink": 0,               // 闪烁状态
    "icon": "Player",         // 图标类型
    "icon_bg": "none",        // 图标背景
    "x": 0.350927,            // 地图X坐标 (0-1)
    "y": 0.358864,            // 地图Y坐标 (0-1)
    "dx": 0.274005,           // 方向向量X
    "dy": 0.961728            // 方向向量Y
}
```

**常见对象类型**：

| type | 说明 |
|------|------|
| `aircraft` | 飞机 |
| `ground_model` | 地面载具 |
| `airfield` | 机场 |
| `bombing_point` | 轰炸点 |

**常见图标类型**：

| icon | 说明 |
|------|------|
| `Player` | 玩家 |
| `Fighter` | 战斗机 |
| `LightTank` | 轻型坦克 |
| `MediumTank` | 中型坦克 |
| `HeavyTank` | 重型坦克 |
| `SPAA` | 自行防空炮 |
| `bombing_point` | 轰炸点 |

### 5.4 `/map_info.json` - 地图元信息

```json
{
    "valid": true,
    "grid_size": [57584.1, 64194.2],   // 地图尺寸（米）
    "grid_steps": [6400.0, 6400.0],    // 网格步长
    "grid_zero": [-24816.1, 31426.2],  // 网格原点
    "map_min": [-32768.0, -32768.0],   // 地图最小坐标
    "map_max": [32768.0, 32768.0],     // 地图最大坐标
    "map_generation": 9,                // 地图版本
    "hud_type": 0                       // HUD类型
}
```

## 6. 开发者进阶：自定义数据

### 6.1 测试极端情况

直接编辑 `script/mock_data.json` 可以测试各种极端场景：

```json
{
    "/state": {
        "valid": true,
        "Ny": 9.5,                    // 测试高G过载告警
        "IAS, km/h": 850,             // 测试超速告警
        "H, m": 15000,                // 测试高空性能
        "water temp 1, C": 130,       // 测试水温过高
        "Mfuel, kg": 50,              // 测试低燃油告警
        "throttle 1, %": 110          // 测试WEP状态
    },
    "/indicators": {
        "valid": true,
        "type": "f-16c",              // 测试喷气机
        "aviahorizon_roll": -60.0     // 测试大坡度滚转
    }
}
```

### 6.2 常用测试场景

| 场景 | 关键字段修改 |
|------|-------------|
| 高G过载告警 | `"Ny": 8.5` 或更高 |
| 超速告警 | `"IAS, km/h": 800+`（根据机型VNE） |
| 失速告警 | `"IAS, km/h": 150`，`"AoA, deg": 15+` |
| 低燃油 | `"Mfuel, kg": 50`，`"Mfuel0, kg": 500` |
| 发动机过热 | `"water temp 1, C": 130+`，`"oil temp 1, C": 100+` |
| 喷气机模式 | `/indicators` 中 `"type": "f-16c"` |
| 活塞机模式 | `/indicators` 中 `"type": "p-51d-20_china"` |

### 6.3 实时修改

Mock 服务会在每次 HTTP 请求时重新读取 `mock_data.json`，因此：

1. 启动 mock 服务：`python script/mock_8111.py serve --port 8112`
2. 启动 VoidMei 并连接到 8112
3. 用文本编辑器修改 `mock_data.json`
4. VoidMei 会在下一次轮询（~50ms）后显示新数据

## 7. 常见问题 (Troubleshooting)

### Q: 启动脚本报错 "Port 8111 is already in use by aces"

**原因**：War Thunder 正在运行，占用了 8111 端口。

**对策**：
1. 使用 8112 端口：`--port 8112`
2. 在 VoidMei 中同步修改端口为 8112

### Q: VoidMei 状态栏一直显示 "等待建立连接"

**检查步骤**：
1. 确认 mock 脚本已成功启动（终端显示 "Serving on port 8112"）
2. 确认 VoidMei 端口配置为 8112
3. 确认 `mock_data.json` 存在且格式正确
4. 尝试在浏览器访问 `http://127.0.0.1:8112/state`

### Q: 数据显示不正确或缺失

**可能原因**：
1. `mock_data.json` 缺少必要字段
2. JSON 格式错误（缺少逗号、引号等）
3. 字段名拼写错误

**验证方法**：
```bash
# 验证 JSON 格式
python -m json.tool script/mock_data.json
```

### Q: 如何获取最新的 mock 数据格式？

运行 `capture` 命令从真实游戏中捕获最新数据：
```bash
# 在游戏运行并飞行时执行
python script/mock_8111.py capture
```

## 8. 与开发流程集成

### 推荐的开发工作流

```
┌─────────────────────────────────────────────────────────────────────┐
│  1. 启动 mock 服务                                                   │
│     $ python script/mock_8111.py serve --port 8112                  │
│                                                                      │
│  2. 启动 VoidMei（连接 8112）                                        │
│     $ java -jar VoidMei.jar                                         │
│                                                                      │
│  3. 修改代码 → 编译 → 重启 VoidMei                                   │
│     $ javac -encoding UTF-8 -d bin -cp 'dep/*' @sources.txt         │
│     $ java -cp 'bin:dep/*' prog.Application                         │
│                                                                      │
│  4. 如需测试特定场景，修改 mock_data.json                            │
│                                                                      │
│  5. 循环迭代直到功能完成                                             │
└─────────────────────────────────────────────────────────────────────┘
```

### 与 IDE 集成

在 IntelliJ IDEA 或 Eclipse 中：
1. 创建运行配置，Program Arguments 设为端口 8112
2. 或在代码中临时硬编码测试端口
